Appendix K Generator

Drop your SRH-2D mesh exports, get plan-view result figures over an aerial — all in your browser. Nothing is uploaded.

Your figure will appear here once you add the two dataset .h5 files and click Generate figure.
0 Naming convention Name your SMS simulations (or the uploaded files) with EX / Existing or PR / Proposed — that's how the app labels each run's condition (and tells the two apart). Without it, the mesh is labeled generically.
1 Data
Mesh files .h5 Two .h5 files, both exported from your mesh in SMS:
File 1 — mesh geometry
  1. Right-click the mesh (under Mesh Data) → Export…
  2. Save as type → XMDF File (*.h5), name it, save.
File 2 — all datasets
  1. Right-click the mesh → Export Datasets…
  2. In Select Tree Item, check Z + the datasets you want (grab all).
  3. Datasets: Shear (B_Stress), Froude, Velocity Magnitude, Velocity (x/y), Water Depth, Water Elevation. Froude not needed.
  4. Name it and save.
Drop the files below — any order; the app detects which is which.
Existing & Proposed
Each mesh needs those two files. For a full Appendix K comparison, upload both your Existing and Proposed meshes (up to 4 files) — the app pairs them by the EX/PR name. You can also load just one condition; the report includes whatever you upload.
Overlays .zip optional Draw shapefiles on the figure — stream centerline (arc), stationing (points), boundaries (polygon). In SMS, right-click the coverage → Export… → Shapefile; zip the parts (.shp/.shx/.dbf/.prj) and drop the .zip below.
